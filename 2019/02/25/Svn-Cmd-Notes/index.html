SVN 命令行

2019-02-25 Git / Svn 阅读量

因为公司使用SVN,一般都是可视化操作,突然想玩一下SVN的命令行

安装

  • 直接点击下载Subversion

SVN可用子命令

常用SVN命令行

import - 上传项目到SVN服务器

注: 服务器上remote_dir若不存在, 会自动创建;
只会上传project_dir目录下的文件到remote_dir的目录下
import之后, project_dir并没有自动转化为工作目录, 需要重新checkout(后面会用到)

checkout - 检出svn项目到本地

add - 添加新文件

update 简写(up) - 更新文件

commit (简写:ci) - 提交到版本库

status - 查看文件或者目录状态

注: svn status、svn diff、svn revert这三条命令在没有网络的情况下也可以执行的，原因是svn在本地的.svn中保留了本地版本的原始拷贝。

delete - 删除文件

log - 查看日志

info - 查看文件详细信息

diff - 比较差异

merge - 合并文件

mkdir - 创建纳入版本控制下的新目录

revert - 回滚

resolved - 解决冲突

branchs - 新建分支

tag - 建立标签

lock - 加锁/解锁

常见问题

不小心提交了一些动态改变根本不需要的文件到svn服务器上
